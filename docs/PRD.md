# PRD 產品需求文件：食譜收藏夾 (Recipe Collection System)

## 1. 專案概述
- **背景與動機**：現代人注重飲食，常在各大社群與網站上看到不錯的食譜，但缺乏一個有效率的地方統一收藏，導致要煮菜時找不到先前的食譜。本系統旨在提供一個簡單易用的食譜統整平台。
- **目標用戶**：食譜使用需求者（包含自炊族、家庭主婦、喜愛料理的人）。
- **核心價值主張**：讓使用者不費力地增加喜歡的食譜，並在需要時能透過分類及搜尋，簡單快速地找出想做的料理。

## 2. 功能需求
系統涵蓋五大核心功能，以下列出各功能的使用者故事：

1. **新增食譜**
   - 作為 **食譜使用需求者**，我希望 **能夠在系統中新增食譜（包含標題、食材、步驟或圖片）**，以便 **把喜歡的料理紀錄下來**。
2. **搜尋食譜**
   - 作為 **食譜使用需求者**，我希望 **能用關鍵字快速搜尋食譜**，以便 **在需要時能夠簡單且快速地找出需求食譜**。
3. **編輯食譜**
   - 作為 **食譜使用需求者**，我希望 **能修改已經新增的食譜內容（例如調整食材比例）**，以便 **根據實際烹飪經驗持續改良我的紀錄**。
4. **分享食譜**
   - 作為 **食譜使用需求者**，我希望 **能夠將儲存的食譜透過連結分享給他人**，以便 **把優質料理和親朋好友交流**。
5. **食譜類別分類**
   - 作為 **食譜使用需求者**，我希望 **可以為食譜建立不同的分類（如：甜點、家常菜、素食）**，以便 **有條理地管理與瀏覽我的收藏夾**。

## 3. 非功能需求
- **技術限制**：
  - 系統後端：使用 Python 搭配 **Flask** 框架。
  - 前端渲染：採用 **Jinja2** 模板，搭配基礎 HTML/CSS/JS (建議支援 RWD 以符合行動端瀏覽)。
  - 資料儲存：使用 **SQLite** 進行資料庫管理（快速輕量，適合 MVP 開發）。
- **效能考量**：確保列表與搜尋頁面的查詢速度，在低延遲下將搜尋結果回傳給用戶。
- **安全考量**：對使用者自訂輸入項目進行驗證避免 XSS 攻擊，並防範 SQL Injection；如有後續會員功能，需保障密碼等個資安全。

## 4. MVP 範圍
- **Must Have (一定要有的核心功能)**：
  - 新增食譜（支援填寫標題、食材清單、步驟說明）
  - 編輯、檢視與刪除食譜 (CRUD)
  - 基本的關鍵字搜尋功能
  - 食譜類別建立與分類顯示
- **Should Have (應該要有，以提供完整體驗)**：
  - 將食譜分享給他人的功能 (例如生成只讀頁面與連結)
- **Nice to Have (有會更好的加分功能)**：
  - 支援食譜圖片上傳
  - 網頁抓取功能 (自動從外部網址截取食譜文字)
  - 會員系統與個人化書籤
